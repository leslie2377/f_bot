## 1. 프로젝트 개요

### 1.1 목적
챗봇에서 발생하는 모든 대화를 영구 저장하고, 관리자가 세션별/화자별로 대화 내역을 조회·검색·분석할 수 있는 관리자 페이지를 구축한다.

### 1.2 현재 문제점
- 대화 히스토리가 Node.js 메모리(Map)에만 저장 → 서버 재시작 시 소실
- 세션 TTL 30분 후 자동 삭제 → 이력 추적 불가
- 어떤 질문이 많은지, AI가 잘못 답한 경우를 파악할 수 없음
- 고객 불만/미해결 건을 추적할 방법 없음

### 1.3 대상 사용자
- 프리텔레콤 상담 운영 관리자
- 고객 서비스 품질 담당자

## 2. 핵심 기능 요구사항

### 2.1 대화 저장 시스템

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|----------|
| S-01 | 대화 영구 저장 | 모든 대화를 JSON 파일(또는 SQLite)에 영구 저장 | P0 |
| S-02 | 세션별 분리 저장 | 세션 ID 기준으로 대화를 그룹화하여 저장 | P0 |
| S-03 | 화자 구분 | 각 메시지에 화자(user/bot) 정보 + 타임스탬프 기록 | P0 |
| S-04 | 메타데이터 기록 | 카테고리, 응답 시간, 토큰 사용량 등 부가 정보 저장 | P1 |

| ID | 기능 | 설명 | 우선순위 |

| D-02 | 세션 상세 보기 | 특정 세션의 전체 대화 내역 (화자별 구분 표시) | P0 |

| D-06 | 대화 내보내기 | CSV/JSON 형식 다운로드 | P2 |

## 6. 기술 스택 (추가 사항)

| 구분 | 기술 | 선택 이유 |
|------|------|-----------|
| **데이터 저장** | JSON 파일 → SQLite (확장 시) | 초기: 별도 DB 없이 시작, 확장 시 SQLite |
| **관리자 인증** | 환경변수 비밀번호 + JWT | 간단하지만 안전한 인증 |
| **관리자 Frontend** | React + Vite (별도 앱) | 채팅 위젯과 독립 배포 |
| **차트** | recharts | 가벼운 React 차트 라이브러리 |
| **날짜 처리** | date-fns | 경량 날짜 포맷팅 |
