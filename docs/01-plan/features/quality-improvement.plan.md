# Plan: 상담 품질 지속 개선 시스템

## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | 대화 영구 저장 + 관리자 모니터링 + 자동 품질 개선 루프 |
| 작성일 | 2026-03-29 |
| 의존성 | f_bot 챗봇 + 관리자 대시보드 (구현 완료) |

### Value Delivered

| 관점 | 설명 |
|------|------|
| **Problem** | 대화 히스토리가 메모리에만 유지, 서버 재시작 시 소실. 미응답/오응답을 체계적으로 추적하고 개선할 방법이 없음 |
| **Solution** | 대화 영구 저장 + 품질 점수 자동 산출 + 미응답→FAQ 자동 전환 루프 |
| **Function UX Effect** | 관리자가 상담 품질을 실시간 모니터링하고, 반복 문의를 자동으로 개선 |
| **Core Value** | 고객 응대율 지속 향상 + 운영 비용 절감 + 데이터 기반 서비스 개선 |

|------|------|-----------|

| ❌ 응답 품질 점수 없음 | 좋은/나쁜 응답 구분 불가 | 자동 품질 스코어링 시스템 |

| ❌ 오응답 탐지 없음 | 잘못된 답변 방치 | 부정 키워드 + 피드백 기반 탐지 |
| ❌ 응답 소스별 분석 없음 | 최적화 방향 모름 | FAQ/캐시/RAG 비율 대시보드 |
| ❌ 일별/주별 품질 추이 없음 | 개선 효과 측정 불가 | 기간별 품질 지표 차트 |
| ❌ 관리자 알림 없음 | 문제 발생 시 인지 지연 | 미해결 건 실시간 알림 |

## 3. 기능 상세 설계

### 3.1 대화 영속성 강화

**문제**: `aiService.js`의 `sessions = new Map()`은 서버 재시작 시 소실

**해결**: conversationStore에 이미 저장된 대화를 AI 히스토리 복원에 활용

```
서버 시작 / 새 세션 요청 시:
  1. Map에 세션 있음? → 그대로 사용
  2. Map에 없음 → conversationStore에서 로드
  3. conversationStore에도 없음 → 새 세션 생성
```

| 변경 파일 | 변경 내용 |
|-----------|-----------|
| aiService.js | getHistory()에서 conversationStore 폴백 로드 추가 |

### 3.2 응답 품질 자동 스코어링

모든 봇 응답에 **자동 품질 점수 (0~100)**를 부여:

```
┌─────────────────────────────────────────────────┐
│              품질 스코어 산출 기준                 │
├─────────────────────────────────────────────────┤
│                                                  │
│  기본 점수: 70점 (RAG 정상 응답 기준)             │
│                                                  │
│  가산 요소 (+)                                   │
│  +15  FAQ 직접 매칭 (검증된 답변)                 │
│  +10  캐시 히트 (이전에 검증된 답변)              │
│  +10  응답에 프리티 링크 포함                     │
│  + 5  응답 길이 적절 (50~500자)                  │
│  + 5  고객 피드백 "좋아요"                        │
│                                                  │
│  감산 요소 (-)                                   │
│  -30  미해결 패턴 감지 ("답변 불가")              │
│  -20  에러 응답 (API 실패)                       │
│  -15  고객 피드백 "싫어요"                        │
│  -10  응답 시간 5초 초과                         │
│  -10  동일 세션에서 같은 질문 반복                │
│  - 5  응답이 너무 짧음 (<30자)                   │
│                                                  │
│  결과: 0~100점 (100=최고, 0=최저)                │
└─────────────────────────────────────────────────┘

품질 등급:
  90~100  ⭐ 우수    (FAQ 직접, 검증 완료)
  70~89   ✅ 양호    (RAG 정상 응답)
  50~69   ⚠️ 주의    (불확실하거나 느린 응답)
  0~49    ❌ 불량    (미해결, 에러, 부정 피드백)
```

| 변경 파일 | 변경 내용 |
|-----------|-----------|
| services/qualityScorer.js | **신규** - 품질 점수 산출 모듈 |
| aiService.js | 응답마다 품질 점수 산출 + 저장 |
| conversationStore.js | 메시지에 qualityScore 필드 추가 |

### 3.4 미해결 → FAQ 자동 전환 루프

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
```

### 3.5 오응답 탐지 시스템

```
오응답 판정 기준:
  1. 고객 👎 피드백
  2. 동일 세션에서 같은 카테고리 질문 반복 (봇 답변 불만족 추정)
  3. "아니요", "그게 아니라", "다시" 등 부정 후속 메시지
  4. 품질 점수 50점 미만 응답

오응답 감지 시:
  → 해당 응답의 캐시 무효화 (잘못된 캐시 제거)
  → 관리자 검토 큐 추가
  → 관리자 수정 → 올바른 답변으로 캐시 교체
```

| 부정 키워드 | 의미 |
|-------------|------|
| "아니요", "아닌데" | 답변 부정 |
| "다시 알려줘", "다시" | 답변 불충분 |
| "그게 아니라", "아니고" | 방향 잘못 |
| "모르겠어요" | 이해 불가 |
| "고객센터", "상담원" | 봇 포기 → 사람 요청 |

## 4. 품질 지표 (KPI)

| KPI | 산출 방법 | 목표 |
|-----|-----------|------|
| **응답률** | (전체 응답 - 미해결) / 전체 응답 × 100 | 95% 이상 |
| **평균 품질 점수** | 전체 응답의 qualityScore 평균 | 80점 이상 |
| **고객 만족도** | 👍 / (👍 + 👎) × 100 | 90% 이상 |
| **FAQ 커버리지** | FAQ 직접 매칭 비율 | 40% 이상 |
| **캐시 히트율** | 캐시 응답 / 전체 응답 × 100 | 30% 이상 |
| **평균 응답 시간** | 전체 응답의 responseTimeMs 평균 | 3초 이내 |
| **미해결 잔여** | 3일 이상 미처리 미해결 건수 | 0건 |
| **주간 FAQ 추가** | 미해결 → FAQ 전환 건수 | 5건+ |

## 6. 필요 사항

### 6.1 기술적 필요 사항

| 항목 | 현재 | 필요 | 이유 |
|------|------|------|------|
